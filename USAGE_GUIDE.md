# Smart Cabinet Pi - 使用流程指南

## 系统架构

```
┌─────────────┐      HTTPS       ┌─────────────┐
│   用户       │ ◄──────────────► │  服务器      │
│  (Web App)   │                  │  (Next.js)   │
└──────┬──────┘                  └──────┬──────┘
       │                                │
       │ 生成配对码                        │ 用户数据
       │                                │
┌──────▼────────────────────────────────▼──────┐
│           Smart Cabinet (Raspberry Pi)         │
│  ┌─────────────┐    ┌─────────────┐           │
│  │  NFC 读卡器  │    │  RFID 扫描器 │           │
│  │  (PN532)    │    │  (RC522x4)  │           │
│  └──────┬──────┘    └──────┬──────┘           │
│         │                   │                  │
│  ┌──────▼───────────────────▼──────┐          │
│  │      Python Controller          │          │
│  │  - 状态机控制 (LOCKED/UNLOCKED)  │          │
│  │  - SQLite 本地数据库             │          │
│  │  - 离线队列同步                  │          │
│  └─────────────────────────────────┘          │
└───────────────────────────────────────────────┘
```

## 使用场景 1: 拍卡借用工具 (正常流程)

### 步骤 1: 系统待机 (LOCKED 状态)
```
┌─────────────────────────────────────┐
│  🔴 系统锁定                          │
│                                     │
│  [请拍卡或扫码]                       │
│                                     │
│  LEDs: 红色 (全部锁定)                │
└─────────────────────────────────────┘
```

**硬件状态:**
- 所有抽屉: 锁定
- LED灯: 红色
- NFC读卡器: 持续监听 (0.5秒间隔)

---

### 步骤 2: 用户拍卡 (AUTHENTICATING 状态)

**用户动作:** 将 NFC 卡靠近读卡器

```
┌─────────────────────────────────────┐
│  🟡 验证中...                         │
│                                     │
│  正在读取卡片...                       │
│                                     │
│  LEDs: 黄色                           │
└─────────────────────────────────────┘
```

**系统处理:**
1. 读取卡片 UID (例如: `CARD-001`)
2. **在线模式:** 调用 `/api/edge/authorize` 验证
3. **离线模式:** 查询本地缓存的用户数据

**认证成功:**
```json
{
  "authorized": true,
  "user_id": "user-123",
  "user_name": "张三",
  "source": "cache"  // 或 "server"
}
```

**认证失败:**
- LED 闪烁红色 + 错误蜂鸣声
- 显示错误信息 (卡片未注册/无权限)
- 3秒后返回锁定状态

---

### 步骤 3: 柜子解锁 (UNLOCKED 状态)

**认证成功:**
```
┌─────────────────────────────────────┐
│  🟢 欢迎, 张三!                       │
│                                     │
│  请取/还工具，完成后再次拍卡关门        │
│                                     │
│  LEDs: 绿色 (全部解锁)                │
│  🔓 抽屉已解锁                        │
└─────────────────────────────────────┘
```

**系统动作:**
1. 播放成功蜂鸣声 (2000Hz, 0.1s)
2. 解锁所有抽屉 (PCA9685 伺服电机)
3. LED 变为绿色
4. **捕获 RFID 开始快照** (记录当前柜内所有工具)

```python
start_rfids = ["RFID-001", "RFID-002", "RFID-003", "RFID-004"]
# Arduino | RPi4   | 万用表 | 电烙铁
```

---

### 步骤 4: 用户取工具

**用户动作:**
1. 打开抽屉 (门磁传感器检测)
2. 取走需要的工具 (例如: Arduino)
3. 关闭抽屉
4. **再次拍卡** 表示完成

```
抽屉 0: 打开 → LED 0 变红色
抽屉 0: 关闭 → LED 0 变绿色

[用户拍卡] → 系统检测到关门命令
```

---

### 步骤 5: 扫描并锁定 (SCANNING 状态)

```
┌─────────────────────────────────────┐
│  🟡 正在扫描...                       │
│                                     │
│  请稍候，正在核对库存...               │
│                                     │
│  LEDs: 黄色                           │
│  🔒 正在锁定...                       │
└─────────────────────────────────────┘
```

**系统动作:**
1. 锁定所有抽屉
2. **捕获 RFID 结束快照**

```python
end_rfids = ["RFID-002", "RFID-003", "RFID-004"]
# RPi4   | 万用表 | 电烙铁
# (Arduino 不见了!)
```

3. **计算差异:**
```python
borrowed = start_rfids - end_rfids
         = ["RFID-001"]  # Arduino 被借走

returned = end_rfids - start_rfids
         = []  # 没有归还
```

4. **显示摘要 (10秒):**
```
┌─────────────────────────────────────┐
│  📋 本次操作摘要                      │
│                                     │
│  借出 (1):                           │
│    ✓ Arduino Uno                    │
│                                     │
│  归还 (0):                           │
│    (无)                              │
│                                     │
│  [离线模式: 已本地保存，将自动同步]      │
└─────────────────────────────────────┘
```

5. **保存到本地数据库:**
   - `session_diffs` 表: 完整会话记录
   - `borrow_history` 表: 借用历史
   - `pending_sync` 表: 待同步队列

---

### 步骤 6: 返回锁定 (LOCKED 状态)

系统自动返回待机状态，等待下一位用户。

## 使用场景 2: NFC 卡配对 (新用户)

### 情况 A: 使用 QR 码配对 (推荐)

**步骤 1: 用户在 Web App 生成配对码**
```
用户登录 Web App → 个人资料 → "绑定新卡片"
              ↓
       显示 6位配对码 和 QR 码
       (例如: ABC123)
       有效期: 5分钟
```

**步骤 2: 用户在柜机拍未配对卡片**
```
┌─────────────────────────────────────┐
│  🟡 卡片未注册                         │
│                                     │
│  请在手机上生成配对码后扫码             │
│                                     │
│  [展示 QR 码]                         │
└─────────────────────────────────────┘
```

**步骤 3: 用户展示 QR 码**
- 系统读取 QR 码内容 (例如: `https://save4223.local/pair?token=ABC123`)
- 提取配对令牌: `ABC123`
- 发送到服务器: `POST /api/edge/pair-card`

**步骤 4: 配对成功**
```
┌─────────────────────────────────────┐
│  🟢 配对成功!                         │
│                                     │
│  卡片已绑定到您的账户                  │
│  现在可以使用此卡开箱                  │
└─────────────────────────────────────┘
```

- 播放成功蜂鸣声
- 缓存用户数据到本地数据库
- 3秒后返回锁定状态

---

### 情况 B: 离线配对 (柜机离线时)

**如果柜机无法连接服务器:**

1. 用户拍未配对卡片
2. 系统提示: "请输入配对码"
3. 用户通过键盘输入 6位数字
4. 系统保存到 `pending_pairings` 队列
5. 显示: "配对请求已保存，将在联网后处理"
6. 恢复网络后，SyncWorker 自动完成配对

## 使用场景 3: 归还工具

与借用流程相同，只是差异计算结果相反:

```python
start_rfids = ["RFID-002", "RFID-003", "RFID-004"]
# 用户归还 Arduino
end_rfids = ["RFID-001", "RFID-002", "RFID-003", "RFID-004"]

borrowed = []  # 没有借出

```

```
│  借出 (0):                           │
│    (无)                              │
│                                     │
│  归还 (1):                           │

```

## 使用场景 4: 离线模式

### 场景: 柜机断网

**系统行为:**
1. 尝试连接服务器失败
2. 切换到 "离线模式"
3. 所有操作继续使用本地缓存

**支持的功能:**
- ✅ 拍卡开箱 (使用缓存的用户数据)
- ✅ 借用/归还工具 (本地 RFID 差异计算)
- ✅ 新卡配对 (排队等待同步)
- ❌ 实时库存查询 (使用本地缓存，可能滞后)

**数据同步 (恢复网络后):**
```python
# SyncWorker 后台线程
每隔 60 秒:
  1. 检查服务器是否在线
  2. 上传 pending_sync 队列中的会话
  3. 同步物品库存缓存
  4. 处理 pending_pairings 配对请求
```

**会话同步优先级:**
1. 高优先级: 借用/归还会话 (影响库存)
2. 低优先级: 配对请求 (可稍后处理)

## 使用场景 5: 同时借用和归还

用户可以在一次会话中既借用又归还:

```python
start_rfids = ["RFID-001", "RFID-002", "RFID-003", "RFID-004"]
# Arduino | RPi4   | 万用表 | 电烙铁

end_rfids = ["RFID-002", "RFID-003", "RFID-005"]
# RPi4   | 万用表 | 示波器 (新归还的)

borrowed = ["RFID-001", "RFID-004"]  # 借走 Arduino 和电烙铁
returned = ["RFID-005"]              # 归还示波器
```

**显示摘要:**
```
┌─────────────────────────────────────┐
│  📋 本次操作摘要                      │
│                                     │
│  借出 (2):                           │
│    ✓ Arduino Uno                    │
│    ✓ 电烙铁                          │
│                                     │
│  归还 (1):                           │
│    ✓ 示波器                          │
└─────────────────────────────────────┘
```

## LED 状态指示

| 颜色 | 含义 | 状态 |
|------|------|------|
| 🔴 红色 | 系统锁定 | LOCKED |
| 🟡 黄色 | 验证中/扫描中 | AUTHENTICATING / SCANNING |
| 🟢 绿色 | 已解锁 | UNLOCKED |
| 🔵 蓝色 | 配对模式 | PAIRING |

**单个抽屉 LED:**
- 绿色: 抽屉已关闭
- 红色: 抽屉打开中

---

## 蜂鸣器提示音

| 声音 | 含义 |
|------|------|
| ♪ (短高音) | 成功/确认 |
| ♪~ (中音) | 警告/提示 |
| ♪♪ (双低音) | 错误/失败 |
